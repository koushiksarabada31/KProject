1.1 What is a Data Structure?

A data structure is a way of organizing, storing, and managing data so that it can be accessed and modified efficiently. The choice of data structure affects the performance of algorithms.

Unit 2: Arrays and Linked Lists

2.1 Arrays

An array is a contiguous block of memory storing elements of the same type.

int arr[5] = {10, 20, 30, 40, 50};
// Access: arr[2] → 30 (O(1))
// Insert at end: O(1) amortized
// Insert at middle: O(n) — shift elements

2.2 Linked Lists

A linked list is a collection of nodes where each node contains data and a reference to the next node.

Key Difference: Arrays have O(1) random access but O(n) insertion. Linked lists have O(1) insertion/deletion at head but O(n) access.
class Node {
    int data;
    Node next;
    Node(int d) { data = d; next = null; }
}

// Singly Linked List Operations:
// - insertAtHead(data): O(1)
// - insertAtTail(data): O(n)
// - deleteNode(key): O(n)
// - search(key): O(n)

Unit 3: Stacks and Queues

3.1 Stack (LIFO)

Last-In-First-Out. Operations: push(), pop(), peek(), isEmpty() — all O(1).

Applications: Expression evaluation, function call stack, undo operations, bracket matching.

3.2 Queue (FIFO)

First-In-First-Out. Operations: enqueue(), dequeue(), front() — all O(1).

Applications: BFS traversal, CPU scheduling, print queue, buffer management.
