🗄️ DBMS Normalization — Complete Notes

Subject: CS204 — Database Systems | Unit: 3

Why Normalization?

Normalization eliminates:

  • Insertion anomaly — Can't add data without other required data
  • Update anomaly — Updating one place but not another
  • Deletion anomaly — Deleting one thing accidentally deletes another

Normal Forms

1NF — First Normal Form

✅ All columns contain atomic (indivisible) values. No repeating groups.

❌ Violation: Student(Name, Phone: "9876, 1234") — Phone has multiple values
✅ Fix: Split into separate rows or a Phone table

2NF — Second Normal Form

✅ In 1NF + No partial dependencies (non-key attribute depends on part of composite key)

// Bad: StudentCourse(StudentID, CourseID, StudentName, Grade)
// StudentName depends only on StudentID, not full key
// Fix: Split into Student(StudentID, StudentName) + Enrollment(StudentID, CourseID, Grade)

3NF — Third Normal Form

✅ In 2NF + No transitive dependencies (A → B → C means A → C transitively)

// Bad: Student(StudentID, DeptID, DeptName)
// StudentID → DeptID → DeptName (transitive!)
// Fix: Student(StudentID, DeptID) + Dept(DeptID, DeptName)

BCNF — Boyce-Codd Normal Form

✅ In 3NF + Every determinant is a candidate key

Quick Reference Table

SQL: Creating Normalized Tables

CREATE TABLE Department (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(100) NOT NULL
);

CREATE TABLE Student (
    student_id INT PRIMARY KEY,
    name VARCHAR(100),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id)
);
